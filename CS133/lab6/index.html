BibTeX

  1. To create a BibTeX reference to an article I can add the following to a .bib file:

    @article{label2,
    author = "Mr BoyManJr",
    title = "The Science of Caring",
    year = "2141",
    journal = "IEEE Transactions",
    volume = "33",
    number = "1",
    pages = "12--52"
    }

    Instead of @artcile in front of the curly braces, different tags can be used such as @book. This is useful as it will change the formatting of the reference when it is cited later on.

  2. I created a number of references in my bibliography.bib file, and then focused on adding these to my LaTeX document. To add the contents of the bibliography to my LaTeX document, I need to import the file using the following command:

    \bibliography{bibliography}

    There are a number of different formatting options, or styles, that can be used to embed the references into the LaTeX document. These are configured using the \bibliographystlye{...} command. A number of different options exist such as plain, ieeetr and alpha to name a few.

    \bibliographystlye{plain} % I went with plain for the purpose of simplicity.

  3. To cite a source, all I need to do is use the \cite{labelname} command as follows:

    According to the works of Einsten \cite{generalRelativityPaper1} there is a fundamental link between energy and mass.

  4. Compiling LaTeX with BibTeX is a little tricky and inconvient. It requires 4 commands:

    pdflatex document % Sets up initial pdf file
    bibtex document % Takes pdf file and generates bibliography
    pdflatex document % Embeds the citations
    pdflatex document % Fixes numbering and other issues

Advanced LaTeX

  1. I learnt about the algorithm and algorithmic packages in LaTeX, each of which need to be added to the preamble using the \usagepackage{...}. algorithmic can be used to embed mathematical algorithms into LaTeX documents. I created a generic for loop in LaTex:

    \begin{algorithmic}
    \STATE $n=10$
    \FOR{i=0; $i<=n$; i++}
    \IF{n Even} \PRINT i \ENDIF \ENDFOR
    \end{algorithmic}

    An interesting feature I learnt about these algorithmic commands is that they are statements themselves, meaning that they can be nested within each other like I have done with the \FOR and \IF statements.

  2. Using the algorithm package, floating algorithms can be added to my LaTeX document which treats them a lot more like embedded figures, allowing them to be captioned and so on.

    \begin{algorithm}
    % Normal algorithm placed here.
    \end{algorithm}

    All that is required to form a floating algorithm is to wrap the contents of the algorithmic environment in a new algorithm environment.

Embedding source code

  1. The listings package can be used to embed source code into LaTeX documents from a number of different programming languages such as Java and Python. Source code is added using the following commands:

    \lstsetlanguage{Java}
    \begin{lstlisting}
    String word = "Apple";
    for (int i = 0; i < word.length; i++){
    if (1%2==0) System.out.println(word[i]);
    }
    \end{lstlisting}

  2. I also learnt that instead of manually typing in the code, the package is powerful enough to read code from a file and add it to the document using the \lstinputlisting{..path..}

  3. The package also includes some interesting customization features where \DeclareFixedFont{...} can be used to set different fonts for different contexts within the source code such as when something should be bold or italic. A more detailed \lstset{...} can also be used to fine tweak settings:

    % Curly brace is used with a number of different parameters this time
    \lstset{
    language=Java,
    basicstyle=\ttm,
    keywordstyle=\ttb\color{{deepred}},
    stringstyle=\color{deepblue},
    frame=tb,
    showstringspaces=false
    }
